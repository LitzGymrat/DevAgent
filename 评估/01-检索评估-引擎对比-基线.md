# 检索引擎比较实验（基线）

## 实验设置

- 18 条代表性 Query，按类型分为 4 组：
  - exact_keyword（包含文件名/函数名）5 条
  - chinese_semantic（纯中文语义）5 条
  - english_semantic（英文语义）3 条
  - mixed（中英/符号混合）5 条
- 检索方式：Dense-only、BM25-only、Hybrid (Dense+BM25+RRF)
- top_k = 8

## 评测结果

| 提问类型 | 引擎 | Hit_Rate | MRR |
|----------|------|----------|-----|
| exact_keyword | dense | 100.0% | 0.900 |
| exact_keyword | hybrid | 100.0% | 0.900 |
| exact_keyword | bm25 | 100.0% | 0.717 |
| chinese_semantic | dense | 100.0% | 1.000 |
| chinese_semantic | hybrid | 100.0% | 0.420 |
| chinese_semantic | bm25 | 20.0% | 0.033 |
| english_semantic | dense | 100.0% | 1.000 |
| english_semantic | hybrid | 100.0% | 0.567 |
| english_semantic | bm25 | 66.7% | 0.381 |
| mixed | dense | 100.0% | 0.900 |
| mixed | hybrid | 100.0% | 0.717 |
| mixed | bm25 | 80.0% | 0.392 |

## 最终全局评测结果 (top_k = 8)

| 引擎 | Hit_Rate | MRR |
|------|----------|-----|
| Dense | 100.0% | **0.944** |
| Hybrid | 100.0% | 0.660 |
| BM25 | 66.7% | 0.381 |

## 结论

- Dense-only 在所有类型问题上的命中率最高，且期望文档几乎总是出现在 rank 1–2，是当前最稳的主力检索方式。
- BM25-only 仅在 exact_keyword / mixed 类型问题上有一定价值，在语义问题上表现较差。
- 当前版本的 Hybrid 在 exact_keyword/mixed 上表现良好，但在纯语义问题上并未明显优于 Dense-only。

**选型决策**：选择 Dense-only 作为默认检索引擎，将 BM25/Hybrid 作为特定场景（如符号/文件名精确查询）的辅助能力。
