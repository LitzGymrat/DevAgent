## 1. 这张评估表到底在说啥？

### 按类型看：

#### exact_keyword（5 个）

> Dense：100% / 0.900  
> Hybrid：100% / 0.900  
> BM25：100% / 0.717

- 三个引擎都能在 top-K 里命中正确文件（100% Hit）。
- Dense / Hybrid 基本都是 rank1 或少数 rank2 → MRR ~0.9。
- BM25 命中位置稍靠后 → MRR 0.717。

→ 对“带 `.py` / `def xxx` 的精确问题”，三种方法都能找到答案，Dense/Hybrid 排得更靠前。

#### chinese_semantic（5 个）

> Dense：100% / 1.000

## 2. 这说明了啥？——Dense 是主炮，Hybrid 是侧舱炮

从这张表里抽出两个非常硬的结论：

### 结论 1：Dense-only 足以作为整个系统的默认检索引擎

- 所有 query 类型上（exact / 中文 / 英文 / mixed），Dense Hit@8 = 100%。
- MRR 接近 1.0，说明期望文件 绝大多数场景下排在 rank1 或 rank2。

你可以很自然地在文档和面试里说：

> “在我们构造的 18 条代表性 query（包含精确关键字、中文语义、英文语义、中英混合）上，Dense-only 的 Hit@8 是 100%，MRR 接近 0.95 —— Dense 在这个项目里足以胜任主力检索引擎。”

这句话含金量非常高：

- 你有 eval 脚本 + 分类样本 + 指标 → 不是拍脑袋；
- Dense-only 表现这么好，本身就是一个结果。

### 结论 2：Hybrid 当前版本**有价值，但“只能当增强/兜底”，不能硬当默认**

- 命中率 Hybrid 也做到了 100%（这是好事）；
- 但 MRR 明显低于 Dense（0.66 vs 0.94），说明很多命中排在了 rank2+ 甚至 rank3+；
- 对中文/英文语义问题尤其明显：Dense 直接 rank1，Hybrid 把它往后挤。

这意味着：

- **Hybrid 没有为“语义型查询”带来额外收益，相反在排序上弱于 Dense-only**；
- 它的意义更像是：
    - 在某些符号/文件名类问题上，BM25 可以带一点信息；
    - 在未来要做 rerank / cross-encoder 时，一开始备用一套候选池；
    - 或者在 Dense 模型失效/切换 backend 时作为 fallback。

```
## 检索引擎比较实验

我们将 18 条代表性 Query 按类型分为 4 组：
- exact_keyword（包含文件名/函数名）5 条
- chinese_semantic（纯中文语义）5 条
- english_semantic（英文语义）3 条
- mixed（中英/符号混合）5 条

对 Dense-only、BM25-only 和 Hybrid (Dense+BM25+RRF) 三种检索方式进行评估，top_k=8，结果如下：

- 全局：
  - Dense:  Hit@8=100%，MRR=0.944
  - Hybrid: Hit@8=100%，MRR=0.660
  - BM25:   Hit@8=66.7%，MRR=0.380

结论：
- Dense-only 在所有类型问题上的命中率最高，且期望文档几乎总是出现在 rank 1–2，是当前最稳的主力检索方式。
- BM25-only 仅在 exact_keyword / mixed 类型问题上有一定价值，在语义问题上表现较差。
- 当前版本的 Hybrid (Dense+BM25+RRF) 在 exact_keyword/mixed 上表现良好，但在纯语义问题上并未明显优于 Dense-only。
因此，我们在系统中选择 Dense-only 作为默认检索引擎，将 BM25/Hybrid 作为特定场景（如符号/文件名精确查询）的辅助能力。
```

### 3.2 在 DevAgent 里把检索策略明确写成规则（而不是神奇的 Hybrid 默认）

比如：

Python

```
def classify_query_type(q: str) -> str:
    if ".py" in q or "def " in q:
        return "exact_keyword"
    # 其它简单规则略...
    return "semantic"

def retrieve(self, question: str, top_k: int = 8) -> List[Document]:
    q_type = classify_query_type(question)

    if q_type == "exact_keyword":
        # 符号类问题，可以考虑 Dense-only 或 Dense+BM25  二选一
        return self.db.search_dense(question, top_k=top_k)
        # 或者：return self.db.search_hybrid(question, top_k=top_k)
    else:
        # 普通语义问题，直接 Dense-only，最稳
        return self.db.search_dense(question, top_k=top_k)
```

你可以再跑一个小小的 sanity check，看路由之后 DevAgent.chat 的表现是否稳定。
