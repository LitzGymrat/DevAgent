## 💼 面试准备清单

### 理解每个模块

- [ ] `src/agent/core.py` - Agent 决策流程
- [ ] `src/rag/vectorstore.py` - 混合检索逻辑
- [ ] `loRA/eval_lora.py` - 微调评估方式
- [ ] `src/agent/tools/*.py` - 各工具实现

### 准备讲述的要点

- [ ] **LoRA 微调**: 为什么选择、效果、参数量
- [ ] **RAG 系统**: 为什么要混合、融合算法、中文精度高的原因
- [ ] **Agent 设计**: 工具路由机制、错误处理、多轮记忆
- [ ] **工程化**: 从设计到部署的完整考虑

### 准备回答的问题

**Q: 为什么用 RRF 融合而不是其他算法？**
A: RRF 对异构排名友好，不需要复杂的参数调优，而且效果稳定...

**Q: 中文语义精度为什么能达到 100%？**
A: 我们使用了业界顶级的向量模型 Text-Embedding-V4，同时针对代码这个特定领域...

**Q: 如何保证工具调用的鲁棒性？**
A: 通过 safe_json_loads 处理各种 JSON 格式错误，同时有完整的参数验证...
